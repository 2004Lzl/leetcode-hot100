# 和为 K 的子数组：从 O(n³) 到 O(n) 的解法演进

## 题目
给定整数数组 `nums` 和整数 `k`，统计数组中**和为 k 的子数组个数**。
子数组是数组中元素的**连续非空序列**。

---

## 解法一：暴力枚举 O(n³)

### 思路
枚举所有可能的子数组起点 `i` 和终点 `j`，对每个子数组求和判断是否等于 `k`。

### 代码
```python
class Solution:
    def subarraySum(self, nums: List[int], k: int) -> int:
        ans = 0
        n = len(nums)

        for i in range(n):              # 枚举起点 O(n)
            for j in range(i, n):       # 枚举终点 O(n)
                # 求和 nums[i:j+1]      # 每次求和 O(n)
                current_sum = 0
                for t in range(i, j + 1):
                    current_sum += nums[t]
                if current_sum == k:
                    ans += 1
        return ans
```

### 问题分析
- **时间复杂度**：O(n³)
  - 外层循环 O(n) × 内层循环 O(n) × 求和 O(n)
- **致命缺陷**：n = 2×10⁴ 时，运算次数约为 8×10¹²，**严重超时**
- **重复计算**：`nums[i:j]` 的和与 `nums[i:j+1]` 的和有大量重叠部分，但被重复计算

---

## 解法二：前缀和优化 O(n²)

### 思路演进
**关键洞察**：子数组 `nums[i:j]` 的和可以表示为两个前缀和的差

```
sum(nums[i:j]) = prefix[j] - prefix[i]
```

其中 `prefix[t]` 表示数组前 `t` 个元素的和（即 `nums[0]` 到 `nums[t-1]`）。

预计算前缀和数组后，任意子数组和可在 O(1) 时间求得。

### 代码
```python
class Solution:
    def subarraySum(self, nums: List[int], k: int) -> int:
        ans = 0
        n = len(nums)

        # 1. 预计算前缀和数组
        front_sum = [0] * (n + 1)       # front_sum[i] = nums[0..i-1] 的和
        for x in range(n):
            front_sum[x + 1] = front_sum[x] + nums[x]

        # 2. 枚举所有子数组，O(1) 查询和
        for i in range(n):              # 子数组起点
            for j in range(i + 1, n + 1):   # 子数组终点（前缀和索引）
                if front_sum[j] - front_sum[i] == k:
                    ans += 1

        return ans
```

### 复杂度分析
- **时间复杂度**：O(n²)
  - 预计算 O(n) + 双重循环 O(n²)
  - 子数组和查询从 O(n) 优化到 O(1)
- **空间复杂度**：O(n) —— 前缀和数组

### 局限性
虽然比暴力法快了很多，但 n = 2×10⁴ 时，O(n²) 仍需约 4×10⁸ 次运算，**仍然超时**。

---

## 解法三：前缀和 + 哈希表 O(n)

### 思路演进
**关键洞察**：固定右端点，寻找左端点

在解法二中，我们枚举了所有 `(i, j)` 对。换个角度：

```
front_sum[j] - front_sum[i] = k
=> front_sum[i] = front_sum[j] - k
```

对于每个右端点 `j`，我们只需要知道：**之前有多少个左端点 `i` 满足 `front_sum[i] = front_sum[j] - k`**。

这正是一个"**查询历史出现次数**"的问题，**哈希表**可以在 O(1) 时间完成。

### 进一步优化：边遍历边统计
不需要先存好所有前缀和，可以**遍历过程中动态维护**哈希表：

```
遍历到位置 i 时：
  1. 计算当前前缀和 front_sum[i]
  2. 查询哈希表：front_sum[i] - k 出现过几次？→ 累加到答案
  3. 将当前前缀和存入哈希表，供后面查询
```

### 为什么需要初始化 `{0: 1}`？
前缀和为 0 代表"一个元素都没取"的状态。没有这个初始值，**从数组开头开始的子数组**会被漏统计。

例如 `nums = [1,2,3], k = 6`：
- 遍历到末尾，前缀和为 6
- 需要找之前前缀和为 `6 - 6 = 0` 的次数
- 初始状态 `{0: 1}` 让这次查询能命中

### 代码
```python
class Solution:
    def subarraySum(self, nums: List[int], k: int) -> int:
        ans = 0
        n = len(nums)

        # 1. 预计算前缀和（也可边算边做，这里分开更清晰）
        front_sum = [0] * (n + 1)
        for x in range(n):
            front_sum[x + 1] = front_sum[x] + nums[x]

        # 2. 哈希表：key=前缀和, value=出现次数
        dic = {0: 1}        # 前缀和 0 出现 1 次（空数组状态）

        for i in range(n + 1):
            target = front_sum[i] - k   # 要找的目标前缀和

            # 查询：之前有多少个前缀和等于 target？
            if target in dic:
                ans += dic[target]

            # 记录当前前缀和，供后面查询
            dic[front_sum[i]] = dic.get(front_sum[i], 0) + 1

        return ans
```

### 复杂度分析
- **时间复杂度**：O(n)
  - 单次遍历，哈希表查询和插入都是 O(1)
- **空间复杂度**：O(n)
  - 哈希表最多存 n+1 个不同的前缀和

### 关键细节
| 要点 | 说明 |
|:---|:---|
| `dic = {0: 1}` | 记录初始状态，确保从数组开头开始的子数组能被统计 |
| 先查询、后记录 | 保证只统计"之前"的前缀和，不把自己算进去 |
| `dic.get(key, 0)` | 安全访问，key 不存在时返回 0 而不是报错 |
| 循环范围 `range(n+1)` | 前缀和数组长度为 n+1，包含 front_sum[0] 到 front_sum[n] |

---

## 三种解法对比

| 解法 | 时间复杂度 | 空间复杂度 | 核心优化 |
|:---|:---|:---|:---|
| 暴力枚举 | O(n³) | O(1) | 无 |
| 前缀和 | O(n²) | O(n) | 预计算前缀和，O(1) 求子数组和 |
| 前缀和 + 哈希 | **O(n)** | O(n) | 用哈希表 O(1) 查询历史前缀和出现次数 |

---

## 思维演进路径

```
暴力 O(n³)
    ↓ 发现重复计算：子数组和有大量重叠
前缀和 O(n²)
    ↓ 发现固定右端点后，只需查询左端点出现次数
哈希优化 O(n)
    ↓ 边遍历边统计，空间换时间
```

**核心思想**：将"枚举所有子数组"转化为"枚举右端点 + 查询左端点"，利用哈希表将查询优化到 O(1)。
